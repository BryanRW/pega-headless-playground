# Project Skills

This file tracks reusable project-specific skills that should be created when class work produces a repeatable process.

## Skill Creation Rule

Create a skill when a class produces a reusable workflow that future agents or contributors should follow consistently. Keep the skill small, concrete, and tied to repository files that actually exist.

Prefer a script or tool instead of a skill when the work is deterministic and can be automated reliably.

## Candidate Skills

These are candidates, not implemented skills:

- Pega DX API adapter setup: how to connect the active web app to real Pega authentication, cases, assignments, customers, and dashboard endpoints.
- Pega metadata rendering workflow: how to turn Pega metadata contracts into React-rendered views, fields, actions, and validations.
- Mobile SDK instance workflow: how to configure and validate SDK-driven mobile app instance experiments.
- Shared package extraction workflow: how to move repeated app logic into `packages/pega-client`, `packages/pega-types`, `packages/mock-data`, or `packages/ui`.

## Current Implemented Skills

None yet.

Do not document a skill as implemented until its files, instructions, and validation path exist in this repository.

## Documentation Standard

Each future skill should state:

- purpose;
- when to use it;
- input files or contracts it depends on;
- exact steps;
- validation;
- what remains intentionally manual or missing.
